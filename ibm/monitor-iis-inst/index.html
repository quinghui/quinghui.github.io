Monitoring IIS installation

Monitoring the process of IIS installation program, you can track log files.

Monitoring IIS installation

In the directory /opt/IBM/InformationServer if Linux, or C:\IBM\InformationServer if Windows
Procedure

  1. On the computer on which the installation program is running, navigate to the ibm_is_logs directory that you specified as the temporary directory. The default temporary directory is /tmp if Linux, or %temp% if Windows.
  2. Open the log file tail -f ISInstall*.log on Linux, or use a text editor that doesn't lock the log file to open on Windows ISInstall*.log is the log file with the most recent date timestamp. Important: Do not delete the installation log files or the log file directory during the installation process.

Monitoring the WAS installation

You can follow the progress of the WebSphere Application Server(WAS) installation by monitoring the installation log files. In the directory /opt/IBM/InformationServer if Linux or C:\IBM\InformationServer if Windows, both directories are default IIS installation path against Linux and Windows. Procedure

  1. Go to the directory cd /opt/IBM/InformationServer/logs on Linux, or cd c:\IBM\InformationServer\logs on Windows
  2. Open the log file tail -f WebSphereInstall*.log on Linux, or using a text to open the log file on Windows Important: Do not delete the installation log files or the log file directory during the installation process.

Monitoring WAS enterprise archive file deployment

You can follow the progress of the WebSphere Application Server installation by monitoring the installation log files. In directory /opt/IBM/InformationServer, that's default IIS installation path for Linux or Windows.

  1. Go to the directory cd /opt/IBM/WebSphere/AppServer/profiles/InfoSphere/logs/server1 on Linux, or cd c:\IBM\WebSphere\AppServer\Profiles\InfoSphere\logs\server1 on Windows
  2. Open the log file tail -f SystemOut*.log on Linux, or using a text editor on Windows Note: Do not delete the installation log files or the log file directory during the installation process.

Monitoring DataStage component installations

You can follow the progress of individual InfoSphere DataStage component installations by monitoring the component installation log files.
In directory /opt/IBM/InformationServer on Linux, or C:\IBM\InformationServer on Windows, that's default IIS installation path.
You can monitor the installation of the following InfoSphere DataStage components:

  • InfoSphere DataStage plug-ins
  • Parallel engine operators
  • Connectors

Each component must be installed into each project on each host system. If you have many projects, the plug-in installation can take considerable time.

Procedure

  1. Go to the directory cd /opt/IBM/InformationServer/Server/StagingArea/Log on Linux, or cd c:\IBM\InformationServer\Server\StagingArea\Log on Windows
  2. Open the log file. tail -f ISInstall*.log on Linux, or using a text editor, open the log file on Windows. Important: Do not delete the installation log files or the log file directory during the installation process.
